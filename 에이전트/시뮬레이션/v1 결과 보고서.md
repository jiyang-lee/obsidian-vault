# 2. v1의 주요 목적

v1은 시뮬레이션 화면에서 운영자가 바로 활용할 수 있는 형태로 결과를 제공하는 것이다.

|항목|내용|
|---|---|
|상황 인식|현재 설비 또는 수요처의 상태를 요약|
|우선순위 판단|위험도나 중요도가 높은 상황을 먼저 확인|
|운영자 행동 지원|확인해야 할 센서, 설비, 조치 방향 제시|
|결과 구조화|`summary`, `action_plan`, `caution` 형태로 응답 정리|
|기록 가능성|필요 시 운영 메모를 DB에 저장할 수 있도록 구성|

# 3. v1 처리 흐름

```
card_id 선택
→ 관련 데이터 조회
→ 센서값 및 모델 신호 정리
→ v1 입력 JSON 생성
→ LLM 호출
→ 운영 판단 결과 생성
→ 시뮬레이션 화면에 출력
→ 필요 시 운영 메모 저장
```
