# 2. v0 핵심 흐름

```
card_id 1개 선택
→ DB 또는 예제 데이터에서 해당 카드 데이터 조회
→ ops_agent_input 생성
→ LLM 호출
→ ops_agent_output 생성
→ 화면 응답 + 필요 시 DB 저장
```

## ops_agent_input

v0는 llm에게 그냥 알아서 판단해줘 하는 게 아니라, 판단에 필요한 내용을 미리 정리해서 넣는다.

|구역|의미|
|---|---|
|`raw_context`|현재 창/window 기준의 실제 센서값, 상위 N개 센서값, feature 정보|
|`priority_context`|이 상황이 얼마나 중요한지, 어떤 카드인지, 모델 신호가 뭔지, 왜 그런지|
|LLM 호출|위 정보를 바탕으로 운영자가 읽을 수 있는 설명 생성|
|`ops_agent_output`|최종 출력 JSON. 요약, 행동계획, 주의사항 포함|

## ops_agent_output

|출력 필드|역할|
|---|---|
|`summary`|현재 상황 요약|
|`action_plan`|운영자가 확인하거나 조치할 내용|
|`caution`|주의사항, 검토 필요 포인트|
|`LLM_OPS_NOTES`|가능하면 DB에 저장되는 운영 메모|
|시뮬레이션 화면 응답|사용자가 화면에서 바로 보는 결과|
